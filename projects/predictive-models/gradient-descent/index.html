Linear and logistic regression, fit by climbing downhill

How it works

This was the fifth-section assignment for my Predictive Analytics (ECON 475) class — a from-scratch implementation of batch gradient descent for both linear and logistic regression. The algorithm starts with arbitrary weights, computes the gradient of the loss function, takes a step in the opposite direction, and repeats until the loss stops dropping.

The same machinery handles two losses. Linear: minimize mean-squared error against the target. Logistic: minimize binary cross-entropy after passing the linear combination through a numerically-stable sigmoid. The convergence speed depends entirely on the learning rate α — too small and you crawl, too big and the loss bounces upward and the safety brake halts you.

The "extra credit" path standardizes the feature first (mean 0, std 1), which puts the loss surface on equal scales across parameters and lets descent converge in under 1,000 steps from a random initialization.

Use cases

Gradient descent is the scaffolding under nearly every modern predictive model. It's how OLS scales beyond the matrix sizes that closed-form solutions can handle, how logistic regression and SVMs are actually fit in production, and — in its mini-batch / Adam variants — how every neural network is trained.

The pieces that matter in practice are nearly always the same as on this page: a sensible learning rate, feature standardization so the loss surface isn't pathologically scaled, and a stop condition that catches divergence before it eats your compute budget.
