ID3 information-gain splits, from scratch + sklearn

How it works

This was the second-section assignment for my Predictive Analytics (ECON 475) class — a from-scratch implementation of the ID3 algorithm. At each split it scores every candidate feature by information gain (the parent node's entropy minus the weighted entropy of the children) and picks the one that reduces uncertainty the most.

The tree is grown to depth 2: the root pick partitions the data once, then each branch picks its own best feature for the second level. Leaves return the majority class of whatever observations land there. The same dataset is then handed to sklearn's DecisionTreeClassifier at depths 2–5 so you can see how additional depth trades off training accuracy against test accuracy on the same 70/30 split.

The dataset is a 1,054-row criminal-justice survey: seven binary descriptive features (marital status, urban residence, union membership, HS graduation, brother is a criminal, prior probation, prior charges) predicting whether the person committed a crime.

Use cases

Decision trees shine when interpretability matters as much as accuracy — credit-risk scoring, medical triage, or any setting where a regulator or domain expert needs to read the rule. They handle mixed feature types natively, need no scaling, and the output is a sequence of human-readable if/then statements.

Their main weakness is high variance: small dataset changes can flip which feature wins at the root. In production they're almost always wrapped in an ensemble — random forests or gradient-boosted trees — that averages many shallow trees to stabilize predictions.
