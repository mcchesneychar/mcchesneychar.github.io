Naive Bayes

Bayesian MAP with feature-independence + Laplace smoothing

How it works

This was the fourth-section assignment for my Predictive Analytics (ECON 475) class. Naive Bayes uses Bayes' theorem to invert P(features | class) into P(class | features), then picks whichever class has the highest posterior probability — a maximum a posteriori (MAP) estimate.

The "naive" assumption is that features are conditionally independent given the class. That's almost always false, but it lets you replace one giant joint probability table (which needs an exponential amount of data to estimate) with a product of small per-feature probabilities. The estimated probabilities are then smoothed via Laplace (add-k) so unseen feature values don't drive a probability to zero.

For continuous features (hours worked, wage, education, work experience), each per-class likelihood is modeled as a Gaussian fit to the within-class mean and standard deviation.

Use cases

Naive Bayes is the workhorse of text classification — spam filters, sentiment analysis, topic tagging — because it scales linearly with both vocabulary size and document count, trains in a single pass over the data, and remains surprisingly competitive even when its independence assumption is violated.

It's also a common first-pass baseline for medical diagnosis and recommender systems. Its weaknesses show up when features are strongly correlated (think pixel intensities in adjacent image regions) or when one needs well-calibrated probability estimates rather than just a top-1 label.
